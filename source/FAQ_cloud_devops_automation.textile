h2(#q01). *Q: Cloudifyはどのような製品ですか。*

 *A:* Cloudifyは、アプリケーションのクラウドへのデプロイと管理、スケーリングを可能にするオープンソースのフレームワークで、私たちはそれをなるべくシンプルにしました。考え方としては簡単で、まずブループリントでアプリケーションのモデリングをします。ブループリントはそのアプリケーションを構成する階層を、各層のインストール/起動/監視方法とともに記述します。ブループリントは、上記すべてを内包する、テキスト設定ファイルの集合体なのです。各Cloudifyデプロイメントには1つ以上のマネージャーがあり、新規アプリケーションの（上記のブループリントを用いた）デプロイメントと、既存アプリケーションの継続的なモニタリング、スケーリング、回復に利用されます。

h2(#q02). *Q: Cloudifyと、Cloud FoundryやOpenShiftのようなPaaSプラットフォームとの違いを教えてください。*

 *A:* Cloudifyを使って従来型のPaaSフレームワークを構築することはできますが、Cloudify自体は従来型PaaSフレームワークではありません。Cloud FoundryやOpenShiftのようなPaaSプラットフォームは共通のアプリケーションスタックとアーキテクチャに主に着目したもので、シンプルな新規アプリケーションの開発とデプロイメントを容易にすることでデベロッパーの生産性を改善するように設計されています。一方、Cloudifyは、複雑で重要なアプリケーションを扱えるように設計されており、そうしたアプリケーションには複数の階層があるのが一般的で、階層もよくあるRubyやNode、Javaのスタックとは限りません。アプリケーションのモデリングにブループリントを利用することで、Cloudifyのユーザーは既存のあらゆるアプリケーションスタックのデプロイメントと管理を自動化することができるのです。これにはまた、アプリケーションスタック自体に対するユーザーの管理を大幅に強化できる利点もあります。一般的なPaaSプラットフォームのように、管理対象が特定のロードバランサー実装の特定バージョンのウェブサーバーに限定される、ということがありません。

h2(#q03). *Q: Cloudifyはサービスとして提供されるのですか。*

 *A:* いいえ、Cloudifyの提供はサービスとしてではありません。任意のクラウドでのインストールと使用を可能にするオープンソースフレームワークとして提供されます。ただ、間もなくトライアルサービスを始める予定ですので、ソフトウェアのダウンロードやインストールをしなくてもオンラインで試すことができます。

h2(#q05A). *Q: Cloudifyと、ChefやPuppet、SaltStack、Ansibleといった構成管理ツールとの違いを教えてください。Cloudifyはこれらのツールと一緒に使えますか。*

 *A:* 構成管理（CM）ツールは、個々のサーバーを構成してサービスに対応させるのに重要なものです。サーバーと望ましい状態を指定されると、サーバーを最終的に望ましい状態にするために必要な全構成手順を確実に踏んでくれます*。* 
 一方、Cloudifyでは、アプリケーションの視点から物事を見ています。つまり、あるアプリケーションスタックについて、その全階層と各階層の依存と詳細といった記述を与えられると、Cloudifyはそのアプリケーションスタックを実現するために必要な全ステップを踏みます。これには、クラウドへのインフラストラクチャーリソースのプロビジョニング（コンピュートとストレージ、ネットワーク）、プロビジョニングされた各VMへの適切な役割の割当、CMの設定（これはCMツールが行うのが通例です）、各階層への適切な情報の注入、それらの正しい順序での起動、その後は各層のインスタンスの継続的なモニタリング、故障時の回復、必要な際には階層のスケーリングも含まれます。各VMの構成に必要となれば、Cloudifyは確かにこれらのCMツールと統合できますし、実際のところユーザーには統合することをベストプラクティスとして推奨もしています。当社のChefと"Puppet":http://docs.getcloudify.org/3.3.1/plugins/puppet/のプラグインや"Script Plugin":http://docs.getcloudify.org/3.3.1/plugins/script/もご確認ください。構成管理のための"Docker":http://getcloudify.org/2014/10/30/Docker-cloud-orchestration-configuration-management.htmlの利用に関する資料や、"Docker Plugin":http://docs.getcloudify.org/3.3.1/plugins/docker/についてもご確認いただけます。

h2(#q06). *Q: オーケストレーションにオートメーション、DevOps、継続的デリバリーなど、最近はいろんな専門用語を耳にします。何を指しているのでしょうか。*

 *A:* オートメーションは特定のタスクに焦点を当てるのが通例です。オーケストレーションは、ステロイドで増強したオートメーションのようなものです。オートメーションのオートメーション　 — つまりそれは、タスクの実行順（例えば、サーバープロビジョニングの際に、ウェブサーバーの前にデータベースを起動させるなど）も考慮に入れた、ワークフローとプロセスのオートメーションであるということです。クラウドオーケストレーションとクラウドオートメーションの比較については良いブログの記事がありますので役立つかもしれません。

DevOpsについて詳しくは、どういったことが"DevOps":http://getcloudify.org/2014/09/28/DevOps-automation-engineer-orchestration.htmlではないのか、について書かれたこの記事をお読みください。しかし真面目な話ですが、開発と運用の壁を取り払い、コミュニケーションを促すのはある意味、心の持ち方の問題です。貴方の業務に直接かかわる場合ですが、必要に応じて開発や運用からタスクを引き受けるのを恐れるものではありません。

h2(#q07). *Q: Cloudifyの前提条件を教えてください。*

 *A:*  For running the Cloudify CLI, you need to have Python 2.7. When running on clouds such as OpenStack or Amazon, you need to have proper IaaS credentials and set them properly in Cloudify. For more details, please see our "guide":http://docs.getcloudify.org.
 Cloudify CLIの動作にはPython 2.7が必要です。OpenStackやAmazonのようなクラウド上で動作する際は、適切なIaaSのクレデンシャルを持ち、それらをCloudifyに正しく設定しなければなりません。詳しくは当社の"ガイド":http://docs.getcloudify.orgをご覧ください。

h2(#q08A). *Q: 結局、Cloudifyはどのクラウドをサポートしているのでしょうか。その仕組みについても教えてください。*

 *A:* Cloudifyはほとんどすべてのクラウドをサポートしています — 本当ですからどうか最後まで聞いてください。出荷時にそのままプラグインも含めて利用できるクラウドがあります。最も一般的なOpenStackやvCloud、Azure CloudStack、Softlayer、AWS、その他多数のクラウドですが、完全な一覧は当社の資料でご確認ください。既存のプロバイダーのないクラウドを利用しようということなら、自分でプロバイダーを書くか（オープンソースは素晴らしいですね）、当社のサポートに連絡することもできます。

h2(#q14). *Q: OpenStackネイティブとはどういう意味ですか。*

Cloudifyは次の2つの主要な指針により、出荷時点でそのまま"OpenStack":http://getcloudify.org/openstack-architecture-cloudify.htmlとネイティブに統合できるように設計されています: 

* OpenStackコアとインフラストラクチャーサービスとの統合。それにはこの完全な"OpenStackコンポーネントリスト":http://getcloudify.org/2014/07/18/openstack-wiki-open-cloud.htmlの項目と、継続的に開発され、リリースごとに追加される項目も通常は含まれます。
* OpenStackの設計指針の踏襲。OpenStackはPythonで記述され、メッセージングやデータベースなどに特定のスタックを使用します。OpenStackネイティブにするためには、OpenStackの設計指針と可能な限り擦り合わせなければなりません。

Cloudify 3.xで私たちはそれらの指針にできる限り沿うために"TOSCA":http://getcloudify.org/openstack-architecture-cloudify.html#TOSCA標準を取り入れ、OpenStackの全コンポーネントをサポートするビルトインプラグインも開発しました。

"OpenStackネイティブ":http://getcloudify.org/2014/09/02/Open-Cloud-OpenStack-Cloud.htmlとすることについて詳しくはこちらをご覧ください。

h2(#q15). *Q: 企業向けのCloudifyとオープンソースエディションの比較になりますが、どう違うのでしょう。*

 *A:* Cloudifyには、"ここ":https://github.com/cloudify-cosmoから展開して、ニーズに合わせてカスタマイズできる永久無料のオープンソースバージョンと、オープンソースエディションの全機能を内包し、その他の利点も併せ持つPremiumエディションがあります。2つのバージョンについては"こちら":http://getcloudify.org/goPro.htmlをご覧ください。詳細が必要な場合には、"お気軽にご連絡":mailto:hello@getcloudify.org?subject=I'd%20like%20to%20hear%20more%20about%20Cloudify%20Premiumください。
