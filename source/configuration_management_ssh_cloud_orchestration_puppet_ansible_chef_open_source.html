O構成管理ツールのためのオーケストレーション

Cloudifyは、コマンドひとつで複数階層のアプリケーションをデプロイするためのモニタリングとロギングを備えたアプリケーションオーケストレーションサービスを提供します。プラグイン可能なアーキテクチャーで構築されたCloudifyでは、あらゆるツールチェーンにプラグインできるため、デプロイ後に適宜必要となる回復やスケールアウト、スケールイン、さらにカスタムポリシーの定義能力も含め、ライフサイクル全体を完全に網羅したアプリケーション管理が可能です。Cloudifyはデプロイメントに関するリアルタイムメトリクスも提供します。


Cloudifyは出荷時点でPuppetやChef、Ansible、その他にも多数の主要なオープンソースツールと統合できるようになっており(サンプルハブを参照)、まだプラグインがなければ自分で簡単に書くことができます。 {% include downloadbuttons.html %}

構成管理ツールとの統合

構成管理ツールは成熟しており、多くの企業で、アプリケーションの迅速なプロビジョニングとより大規模なデプロイメントに活用されています。しかしながら、構成管理ツールはまだデプロイ前のVMプロビジョニングや、モニタリング、回復、スケーリングといった人手による作業を必要とします。また構成管理ツールは、多数のコンポーネントを、アプリケーショントポロジーのインテリジェントな観測のないまま組み込んでいます。

これが、アプリケーションライフルサイクル全体のオーケストレーションと管理に、Cloudifyが有効である理由です。

オーケストレーションのプロセス

以下は、AnsibleやPuppet、Chefを対象としたCloudifyのオーケストレーションの基本フローです:

ステップ1: ブループリントを用意してアップロードする

CloudifyがサーバーのプロビジョニングとVM作成、エージェントをインストールする

ステップ2: アプリケーションをインストールする

ChefやPuppet、Ansible、Docker、SaltStackと統合して多彩なアプリケーションサービスをインストールする

ステップ3: アプリケーションを管理する

モニタリングやロギング、アラートを追加し、アップデートやパッチなどのワークフローの実行する

ステップ4: 自動回復と自動スケーリングを追加する

継続的にエラーを検知して修正し、キャパシティを追加する

デプロイメント機能


サーバーとエージェントのインストール

Cloudifyはアプリケーショントポロジーをインプットとして捉え、当該アプリケーショントポロジーを基にマシンをプロビジョニングします。Cloudifyでは構成管理クライアントをマシンごとにインストールできます。サーバーをプロビジョニングしたり、全サービスを既存のサーバーにリンクするのみといった場合もあります。環境が設定されると、Cloudify制御コンソールからリモートコマンドを直接呼び出し、必要に応じて簡単に新規のモジュール/クックブックをインストールしたり、デプロイメントを管理したりすることができます。


既存の構成管理レシピ/エージェントのサポート

構成管理ユーザーの多くは、既にエージェント/レシピと構成の作成に投資してきています。Cloudifyは既存の環境上にサービスとして設定されるように設計されているため、システム全体の再構成を強いるよりは既存の環境を活用できるようにします。


アプリケーションライフサイクルの管理

複数の層からなるアプリケーションのデプロイメントには、相互に依存するウェブコンテナやデータベースといった複数のサービスが含まれます。Cloudifyなら、これらのサービスを正しい順序で開始させることができます。各アプリケーションのライフサイクルを監視し、各サービスへのリファレンスを注入するフックを内包します。以下の例では、データベースへのレファレンスを各ウェブコンテナに注入し、ウェブコンテナの各レファレンスをロードバランサーサービスに注入しています。

デプロイ後の機能


モニタリングとロギング

デプロイメントにおける課題のひとつは、構成管理システムと監視システムの同期を維持することです。

Cloudifyではデプロイメントプロセスの一環として自動的にアップデートされる監視機能を搭載することで両システムを結んでいます。また、Cloudifyによるモニタリングでは、他の監視システムではあまりカバーされていない空白を埋めることができます。デプロイメントとセットアッププロセスも監視が必要なところを、他の監視システムはデプロイ後だけを監視するように設計されているためです。加えてCloudifyでは、ロギングとアラートを搭載し、各マネージドサービスに特有のKPI測定に必要なカスタムメトリクスも提供します。


遠隔実行

DevOpsユーザーにとって重要な特長のひとつは、既存のデプロイ済みのサービスとやりとりし、新規アップデートをプッシュできる機能です。Cloudifyでは、DevOpsデプロイメントプロセスの一環として、リモートシェルや構成管理ツールのコマンドを呼び出せる遠隔実行コマンドを追加しています。この遠隔実行機能により、特定のサービスの単一またはすべてのインスタンスにこの呼び出しを簡単に設定できます。

クラウドのポータビリティ

Cloudifyは、複数のプライベートおよびパブリッククラウド環境だけでなく従来のデータセンターにも対応するように設計されました。ChefやPuppet、Ansibleのユーザーは（他の構成管理ユーザーも）、この機能を活用してその環境のアプリケーションを要約することにより、同一のクックブック/モジュールとCloudifyサービスのセットを様々なクラウドでそのまま利用することができます。

高度なオーケストレーション

Cloudifyは、各サービスと交流し、適切なコマンドを実行することで複雑なプロセスを自動化できるオーケストレーションサービスを備えています。

以下を扱う内蔵プロセスのセットが備わっています:

  • 自己回復
  • 自動スケーリング
  • 障害復旧の自動化
  • DevOpsポリシー（アップデートのプッシュなど）
