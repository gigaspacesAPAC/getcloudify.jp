<strong>

h1. Cloudifyについて

</strong>

<p>Cloudifyは、オープンソースでTOSCAベースのクラウドオーケストレーションソフトウェアプラットフォーム（アマゾンのOpsWorksのようなもの）です。アプリケーションスタックのインストールやデプロイメントのプロセスに加えて、モニタリングや改善、自動スケーリングといったデプロイ後のプロセスも自動化します。</p>

<strong>

h2. 実際の動作

</strong>

<p>Cloudifyは人間のオペレーターの作業に似た手順を踏む「ソフトウェアのオペレーター」として動作しますが、すべて自動化されています。このレベルの自動化を達成するため、Cloudifyはブループリントとして知られるソフトウェア構成ファイルで解釈される、インプットを受領します。そのブループリントは、望ましいブループリントの構成を実行するための、アプリケーションとデータセンター間のAPIを介したインタラクションを定義しています。</p>
<p>以下はCloudifyを利用して、アプリケーションをデプロイおよび管理する際の一般的な手順です:</p>

* TOSCAベースのブループリントでアプリケーションを描写する: レシピでは、アプリケーションのコンポーネントとそれらの依存性、バイナリの場所や、インストレーション、モニタリングの構成といった、アプリケーションのインストールと管理に必要な詳細を含むアプリケーションのブループリントを記述します。
 <br />
* 計画の実行: Cloudifyはブループリントを、デプロイメント計画を記述し様々なクラウド環境上で人間のオペレーターのようにその計画を実行するための、インプットとして利用します。 
 <br />

<p>一般的にデプロイメントでは以下の手順を踏みます:</p>


## インフラストラクチャーを設定する: ネットワーク構成（ルーター、セキュリティグループ、パブリックIP）を設定し、コンピュートとストレージのリソースも設定、続いてそれらを関連するネットワークとストレージ機器に割り当てる。
## 各マシンに役割を割り当て、SSHかシェルスクリプト、またはChefやPuppetのような構成管理ツールを呼び出して、それらのマシン上に関連するアプリケーション層をインストールする。
関連サービスをそれらの依存性に基づいた正しい順序で開始する。

## 関連サービスをそれらの依存性に基づいた正しい順序で開始する。
## アプリケーションのモニタリングとロギングを関連付ける。

* アプリケーションを管理する: アプリケーションがデプロイされると、Cloudifyはアプリケーションが期待されるSLAを満たすか確認するために継続的に監視し、障害かスケーリングイベントかに関わらずSLA違反の場合には是正策を取る。<br/>

<strong>

h2. Cloudify上で動作する一般的なアプリケーション

</strong>


<p>Cloudifyは、マニュアルのプロセスでは管理しにくい複数階層のアプリケーションデプロイメントに最もよく適しています。そのようなアプリケーションには以下が挙げられます:
</p>

* 複数階層のJEEアプリケーション（WebLogic、WebSphere、Tomcat、JBoss）
* ビッグデータのアプリケーション
* ネットワークサービス（NFV）

<strong>

h2. サポート対象のクラウドプラットフォーム

</strong>

<p>CloudifyはOpenStackと緊密に統合されていますが、同時に他のプラットフォームもサポートしており、あらゆるクラウドプラットフォームとの相互操作性を実現しています。以下は、Cloudifyがサポートする、ポピュラーなクラウドプラットフォームの一覧です:
</p>

* OpenStack 
* AWS
* CloudStack
* Microsoft Azure
* VMWare 

<strong>

h2. 非クラウドプラットフォームのサポート（ベアメタル）

</strong>

<p>Cloudifyは、マシンの既定プールにプラグインし、そのプールのバウンダリーの範囲でアプリケーションデプロイメントを管理することで、非クラウド環境もサポートしています。この構成はクラウドベースの環境を持たない企業や、クラウドに移行中の企業の間で人気があります。ビッグデータアプリケーションやネットワーク機能のような、I/Oインテンシブでパフォーマンスの集約的なアプリケーションへの対策としても人気です。</p>

<strong>

h2. 構成管理フレームワークのサポート

</strong>

<p>今日では多くの企業が、アプリケーション設定の記述にChefやPuppetなどの構成管理ツールを利用しています。Cloudifyはこれらのフレームワークと統合し、その上にオーケストレーション機能を追加します。ChefやPuppetのユーザーはこの方法で依存性管理やモニタリング、自動回復、自動スケーリングを既存の環境に追加することができます。</p>

<strong>

h2. Cloudifyと他の業界標準やテクノロジー動向との関係性<br/>

h3. TOSCA 

</strong>

<p>Cloudifyでは、Oasis FoundationによるTOSCA（クラウドアプリケーションのためのトポロジーとオーケストレーションの仕様）をブループリントの標準仕様として用いています。</p>

<strong>

h3. DevOps

</strong>

<p>DevOpsは、開発段階から制作段階までのソフトウェアリリースプロセスを自動化するモデルです。
Cloudifyなどのオーケストレーションエンジンは、新規ソフトウェアをパッケージ化し、適切なノードに送り、関連するテストなどを実行するために必要な手順をオーケストレートすることにより、継続的なデリバリーとデプロイメントのプロセスの自動化に貢献します。
</p>

<strong>

h3. NFV

</strong>

<p>NFV（ネットワーク機能仮想化）とは、通信キャリアが、自社の既存のアプライアンスベースのインフラストラクチャーをクラウドベースのインフラストラクチャーに変容させるために推進している構想です。Cloudifyなどのオーケストレーションエンジンは、キャリアのインフラストラクチャーへの仮想ネットワーク機能のプロビジョニングプロセスを自動化し、それらのネットワークサービスによる要求SLAの充足を確保することにより、この移行に重要な役割を果たします。Cloudifyは、エージェントレスなデプロイメントのサポートや、ベアメタルのサポート、ネットワークオーケストレーションの搭載、フェデレーテッドデプロイメントなどを通じてネットワーク機能を管理するように設計されました。</p>

<strong>

h1. GigaSpaces Technologiesについて

</strong>

<p><a href="http://www.gigaspaces.com">GigaSpaces Technologies</a> は、2000年に設立され、急速に成長している革新的なテクノロジー企業、本社はイスラエルに、オフィスはニューヨークや、サンノゼ、ロンドン、香港など世界中に設けています。当社は世界有数の大企業との取引を通じて、年を追うごとに大きく成長を続けています。</p>

h2. 製品とテクノロジー

<p>GigaSpacesは、XAP（Extreme Application Platform）そしてCloudifyという2つの主要な製品ラインを通じてクラウド環境上でのミッションクリティカルなアプリケーションのデプロイメントと、管理、スケーリングのためのソフトウェアやミドルウェアを提供しています。</p>
<p>XAPは、高速なイベント処理とデータアクセスを活用するインメモリコンピューティングプラットフォームで、極度のスケーラビリティをもたらし、アプリケーションによる膨大なデータのリアルタイムでの処理を実現します。XAPでは、ハードウェアを変えずにアプリケーションのパフォーマンスを増強し、可用性と稼働時間を改善するとともに、ライセンスとハードウェアのコストを最小限に抑えることができます。</p>
<p>Cloudifyはオープンソースソフトウェアプラットフォームであり、企業は既存または新規の企業向けアプリケーションを、どのクラウドにも、コードやアーキテクチャーを全く変えずに、迅速にデプロイ、管理、スケーリングすることが可能です。Cloudifyは、あらゆるクラウドとアプリケーションスタックに、クラウドオーケストレーションとDevOpsオートメーションをもたらします。</p>
