Cloudify ピュアプレイオーケストレーションの導入

Cloudify によるアプリケーションライフサイクルの完全なオーケストレーション

テクノロジーアプリケーションスタック、ツール、インフラストラクチャーの多様化する現状では、自社のビジネスやデベロッパー、チームに有効なものを組み合わせるのも大変な作業です。

ここでもし、スタックもツールも自由に選び、ハイブリッド環境も含めたどのようなインフラストラクチャーでもそれらを簡単に運用できるとしたらどうでしょう。コンフィギュレーションはゼロで簡単に、です。

TOSCA（クラウドアプリケーションのためのトポロジーとオーケストレーションの仕様書）に準拠するCloudifyはテクノロジーにとらわれないため、トポロジーのモデリングが終われば、アプリケーションライフサイクル全体のオーケストレーションが可能となり、さらに全体の管理と監視、スケーリング、回復を同一の拠点から行いながら、任意のインフラストラクチャーへのデプロイメントを実施します。

また、Cloudifyと、先進のクラウド、インフラストラクチャー、ツール（実にOpenStakからVMware、Chefを用いたDocker、PuppetやFabricのあるSSHまで）との常識を超えた統合により、文字通りダウンロードするだけで始められるのです。

クラウドアプリケーションのピュアプレイオーケストレーション

大半のツールはアプリケーション管理のインストールと設定の段階に焦点を当てていますが、実際に管理作業が発生するのはアプリケーションのデプロイ後ということが多いものです。

イベント駆動型ワークフローやTOSCAのモデリング言語により、どのツール、環境、ワークロードでもプラグインし、その上に自動プロセスのレイヤー（デプロイメントからポストデプロイメント、自動スケーリング）を、さらにはモニタリングやメトリクスまでも追加することが可能になりました。これにより、Cloudifyはオーケストレーションとモニタリングの間の境界を無くし、既定のイベントに適切な修正措置で自動対応するメカニズムを提供します。修正措置の自動トリガーに対するモニタリングとカスタムのポリシーもこれに含まれ、インテリジェントな自動回復と自動スケーリングの機能を実現します。


すべてのユースケース用のOpenStackオーケストレーション

 

OpenStack Orchestration for Any Use Case

CloudifyはOpenStackネイティブなので、あらゆる種類のOpenStackデプロイメントとシームレスに連携することができ、ユースケースに対して簡単にアプリケーションを編成できます： ハイブリッドクラウド、NFV、データ資産の移行作業等。

Cloudifyは、NeutronからMuranoまでのすべてのOpenStack APIに対応しております。

CloudifyのMuranoはTOSCAの設計図の管理とオーケストレーションをHorizonダッシュボードから直接サポート致します。 Muranoのインテグレーションに関するブログを読む

すべてのVMware stack用のTOSCAオーケストレーション

VMware-Cloudifyとの統合により、ユーザーはインストール、デプロイ、デプロイ後のプロセス（モニタリング、自己修復、自動スケーリング）を自動化にするフレームワークを利用できるようになりました。VMwareのユーザーは、Cloudifyで、VMware環境を広げるためにOpenStackなどの他クラウドを使い、ハイブリッドクラウドを形成することやNFVオーケストレーションを作成することもできます。

ハイブリッドワークロード用のDockerオーケストレーション

CloudifyにはDockerのプラグインとマッチングタイプが導入されており、ユーザーはDockersコンテナをトポロジーとしてご利用できます。これにより、Dockerベースのシステムのための本格的なオーケストレーションと自動化メソッドをご提供できます。CloudifyはKubernetesの内蔵プラグインを保有しています。さらにCloudilyはTOSCAのエンジンにより、Kubernetesが簡単にTOSCAのクラウドを拡張、同じテンプレート内の非マイクロサービスと一緒にマイクロサービスを融合し、さらにクロスクラウド、環境、ワークロードオーケストレーション全体を提供するシステムのなかで新たな言語を作成しました。

どのインフラストラクチャーに対応のプラグイン

SoftLayer, Apache CloudStack, AWS, Azure & Digital Ocean用のプラグインがCloudifyに内蔵されております。プラグイン可能なアーキテクチャが同じ設計図を使用し同じアプリケーションを複数のクラウドで使うことが可能になりました。こちらは、既存の環境からクラウド環境へ移る利用者にとってとても便利な機能です。また、Openstack,VMware,Amazon等、ほかのクラウドにおけるバーストとハイブリッドクラウドを利用可能な状態に致します。

Cloudifyとハイブリッドクラウド＆ワークロード

現実は様々なインフラで様々なワークロードを実行されています（コンテナからクラスタ・データベースやベアメタル上のビッグデーターやアナリティクスシステムなど）ハイブリッド環境とワークロードを接続するには、オーケストレーションのプラットフォームが必要です。CloudifyはTOSCAのエンジンによって複雑なシステムに対して単一画面を作りました。


Cloudifyは企業のクラウド移行と移植性をシンプルに

最近の多くの企業は異種の内部ITインフラストラクチャーに悩まされています。様々な技術を管理することはアライメント、バージョン管理、保守性 、リソース、効率の低下などが問題となっています。

企業は様々なツール、レガシー、OpenStackからVMwareへのデータセンターなどをサポートする一つのオーケストレーションのプラットフォームが必要です。
