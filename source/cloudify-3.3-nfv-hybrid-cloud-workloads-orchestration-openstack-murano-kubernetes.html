Introducing Cloudify 3.3 - More NFV, Multi Cloud & Mixed Workloads, Better Security

You can read about everything that went into bringing you Cloudify 3.3, and let us know what you think. Below is a quick snapshot of the most important updates.

    Introducing the Cloudify Composer



Say hello to the GA of the TOSCA Blueprint drag & drop composer. Previously available as a technology preview, this is our first release of the graphical blueprint composer. It allows complete modeling of complex applications and systems, including networking, compute resources, software infrastructure and application modules. With support for uploading additional artifacts, defining inline types and even new TOSCA interfaces, this composer will make the modeling of even the most complex systems much more straightforward.

What's New in This Version

Back to Basics with CentOS / RHEL 7 Native Componentized Manager

With the previous version of Cloudify, all manager components were packaged into a single Docker container. This provided some benefits in terms of ease of installation and packaging, but turned out to be restrictive and not flexible enough for a lot of our users. In 3.3, we’ve decided to get back to basics and implement a native CentOS / RHEL manager blueprint (we’ve seen that the vast majority of our enterprise Linux users standardized on CentOS and RHEL). In the new manager blueprint, every piece of the Cloudify manager’s stack is modeled as a different Node in the blueprint, which makes it a lot more transparent and extensible than the previous Docker based installation. For example, you can easily replaced the stock ElasticSearch setup with your own ElasticSearch cluster.

Extreme Scale - Tried & Tested

The Cloudify manager can now handle thousands of deployments on a single server. We’ve reached as far as 8000 in our tests, and we’re sure we can push the limits even further.

Just Plain Old Better Security

As with any enterprise product, security is a key element in Cloudify. In version 3.3 we’ve further enhanced it with support for pluggable authorization, allowing users to define roles and access privileges to all REST API resources.

Improved Web UI

We’ve added a slew of usability improvements, starting from much faster page load times, live-searchable tables and lists, and a whole new graphical topology view based on the work we’ve done with the blueprint composer.

Better User Experience of Installation and Setup

We’ve invested a lot in making the setup of the manager and Cloudify agents more robust and less fragile. The agents can now be installed via cloud-init instead of SSH, and in such case no SSH connection is ever initiated between the agents and the manager. In addition, Cloudify 3.3 manager modules are now packaged as RPMs, and all plugins are stored in the built-in repository. This means that the installation can now be done entirely offline without the need to access the internet.

API Enhancements

This version focused on all aspects of better usability, and enhancing key aspects related to our API - starting with pagination of all types of list-based operations and large data sets that are returned from the server. Our API is also now fully versioned, and even if it changes moving forward, it will be versioned, which will prevent any breakage with previous versions of our API.
