VMware NFVとハイブリッドクラウドのためのCloudify

NFVとオペレーターサービスのためのCloudify Telco Edition™ +とVMware vCloud NFV™ +の組合せ

ネットワーク機能仮想化は、通信キャリアが既存のアプライアンスベースのインフラストラクチャーを、クラウドベースのインフラストラクチャーに変容させるために推進している構想です。市場に出すまでのスピードを加速しながらも運用コストは下げる要求が高まる状況下で、通信事業者は、業界の求めるスピードと安定性をもって仮想ネットワークのワークロードをデプロイ、オーケストレイトし、維持するといったNFVクラウドを立ち上げられるソリューションを求めています。

Read all about the VMware/Cloudify Collaboration  read now    

VMware vCloud NFV™ でCloudify Telco Edition™ のオーケストレーションとVNF管理を業界で最高のNFVIプラットフォームと組み合わせることこそ、今日の通信事業者が探している答えです。Cloudify Telco Edition™の持つ強力なMANOとVNFMの機能を、VMware vCloud NFV™オペレーターサービスプロバイダーの提供する堅牢な仮想管理と併せて活用することで、彼らの顧客に敏捷性と安定性を備えたソリューションを容易に提供することができます。

統合プラットフォームの主な特長には以下があります:

  • VNFと全サービスチェーン向けのTOSCAブループリント
  • サービスカタログ
  • vCloud Director™とVMware Integrated Openstack™ デプロイメントの両方を対象にした、同一オーケストレーションエンジンからの複数VIMサポート
  • vRealize Operations Manager™とvRealize Log Insights™による統合されたオペレーションアラートとキャパシティプラニング
  • 完全なソフトウェア定義のネットワーキングソリューションを目的としたNSX™との緊密な統合
  • ETSI準拠のアーキテクチャー

コンバージドITおよびオペレーターサービスのためのCloudify Telco Edition™とVMware vRealize Automation™の組合せ

今日の通信事業者は、OSSやBSSのような従来型のITワークロードが、オペレーターサービスと同一のクラウドインフラストラクチャーを共有するOne Cloud（ワンクラウド）戦略に向かっていますが、それらの機能のオーケストレーションや管理、自動化の実現という課題に直面しています。このソリューションを、一般的なハードウェアプラットフォームにある上記と組み合わせることにより、現代のオペレーターサービスプロバイダーは内部ITワークロードをサポートし、かつ彼らの顧客が求める安定的で堅牢なサービスの提供も続けながら、標準化されたアーキテクチャーの柔軟性を得ることができ、そのアーキテクチャーのコスト節減も実現できます。

Cloudify™のVstrong>VMware vRealize Automation™とvRealize Suite™との組み合わせなら、Cloudifyのモデリングの能力を活用しながらもvRealizeの提供する有名なサービスカタログのインターフェイスを手にすることができます。vRealize Operations Manager™やLog Insights™と緊密に統合されることで、オペレーターは単一の画面からクラウドの運用状況を把握し、問題が発生する前に課題を特定できるようになります。

統合プラットフォームの主な特長には以下があります:

  • 使いやすいサービスビルダー
  • チャージバック/ショーバック
  • vRealize Operations Manager™とvRealize Log Insights™による統合されたオペレーションアラートとキャパシティプラニング
  • 完全なソフトウェア定義のネットワーキングソリューションを目的としたNSX™との緊密な統合
  • ハードウェアインフラストラクチャーの利用を最大限に効率化するための優先順位付けを伴ったワークロードの自動スケールイン/スケールアウト

VMware向けのハイブリッドクラウドのサポート

三つのVMware環境をすべてサポートすることで、VMwareユーザーに共通の管理と自動化の枠組みをもたらします。これにより、vCloudAir でVMwareハイブリッドクラウドをパブリッククラウドとして、そしてvSphere/vCenterやOpenStackもプライベートクラウドとして容易に構築できるようになります。動画でデモをご覧ください。

Cloudifyではシンプルなスクリプトベースのパッケージに始まり、Chef、Puppet、SaltStack、今ではDockerまで広範なアプリケーションパッケージと構成管理をサポートしています。これによりVMwareユーザーはこれらのフレームワークにしてきた投資を活用し、Cloudifyは既存の環境を補完するプラットフォームとして含めることができるのです。


ソリューションの主な利点

  • VMwareのハイブリッドデプロイメントに対するTOSCAベースのオーケストレーション
  • アプリケーションのライフサイクル全体を通じたトポロジー駆動型の管理と監視
  • あらゆるツールチェーンとの統合

Cloudifyの差別化要因

Cloudifyは、多様な企業や通信事業者の環境に適合する唯一のピュアプレイのオーケストレーションフレームワークというユニークなポジションにあり、以下の特長を備えています:

  • OpenStackネイティブ対応を伴う、あらゆるアプリケーション スタックのサポート
  • コンテナ化/非コンテナ化されたワークロードのサポート
  • 標準のTOSCAブループリントを使用したデプロイメントモデリング
  • 包括的モニタリングのサポート
  • デプロイ済みのアプリケーションに対する自己回復と自動スケーリングポリシーを搭載
  • DevOps関連ツールチェーンのサポート

ユースケース

Cloudifyは以下を対象とするオーケストレーションフレームワークとして機能しながら様々な役割を果たします:

  • クラウドオーケストレーション
  • SaaSの利用
  • ネットワーク機能仮想化（NFV）
  • プライベートPaaS
  • ディザスタリカバリ
  • ビッグデータ管理
  • マーケットプレイスフレームワーク
